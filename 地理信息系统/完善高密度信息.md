# 完善高密度信息（并集增强版）

1. introduction：GIS（Geographic Information System）是用于采集、存储、管理、查询、分析、可视化与共享地理空间数据的系统，地理空间数据同时描述位置与属性；发展脉络可记为2017静态到实时、2018从2D/3D到高清、2021从3D GIS到元宇宙（现实到虚拟）；五大组成=硬件（服务器/桌面/云/终端显示设备）+软件方法（GUI、空间分析工具、DBMS）+数据（空间数据与属性数据）+人员（运维和分析决策）+组织（制度流程与业务协同）；两类坐标系统=地理坐标系（经纬度）与投影坐标系（x,y），两类数据表达=矢量与栅格；GIS界面核心媒介是地图，制图设计五原则是视觉对比、易读性、图底组织、层次组织、平衡；数据探索两主线是基于地图（分类、聚合、对比）和基于要素（DBMS属性查询）；平台形态包含Web GIS和移动GIS；可记总式GIS=Map+Database，Map可含Vector+Raster+TIN。

2. coordination system：叠加分析前提是图层空间对齐，必须统一空间参考（坐标系+基准面+投影）；地球几何近似椭球，地理坐标基于椭球定义（大地坐标），大地水准面是仅受重力与地球自转影响的平均海平面延拓；地理基准（Datum）是定义椭球参数、原点、定向的数学参考框架，不可简单等同“以地心为中心的基准面”；ECEF是地心地固直角坐标系（地心笛卡尔），ENU是局部切平面坐标系（非地心）；地图投影是三维曲面到二维平面的数学映射，无法实现全局无变形，只能在形状/面积/距离/方向间权衡；关键参数含主比例尺principal scale、比例尺scale、比例因子scale factor、标准线standard line、中央经线/中央纬线central lines；常用投影有横轴墨卡托Transverse Mercator、等距圆锥Equidistant Conic、Web Mercator，常用投影坐标系统是UTM（60个6度带，中央经线比例因子小于1，常用k0=0.9996）；在GIS中地图可缩放显示，但原始测图精度不因放大而提升。

3. vector data mode：矢量模型以点（0D）线（1D）面（2D）表达离散地物，强调几何精确边界与拓扑关系；数据模型本质是空间对象及其关系，可形式化为有向图并用邻接矩阵、关联/入射矩阵描述连通关系；拓扑价值在于保证数据质量与完整性、支撑网络分析、支撑空间查询与叠加分析，但并非“数据结构简单”；地理关系数据模型通常几何与属性分离存储（几何文件+关系表）；覆盖型拓扑核心是连通性（弧在节点连接）、区域定义（弧环成面）、连续性/方向性（弧有方向及左右多边形）；面向对象地理数据库（如ESRI ArcObjects/Geodatabase）强调依赖、关联、聚合、组合、继承与接口机制，优势是层次组织清晰、可扩展、可动态拓扑、便于行业对象模板化；复合要素典型是TIN/Region/Route，Polygon本身属于基本面要素而非复合要素；TIN属于矢量地形模型，地形起伏大区域应加密采样并倾向小三角形，平坦区可用较大三角形；Shapefile一般为非拓扑简单要素存储；同一图层Polygon通常不应重叠（按无缝分区建模约束）。

4. raster data mode：栅格模型以规则像元阵列表达连续空间现象（降水、温度、高程、土壤等），每个像元记录类别或数值，属性与位置天然绑定；关键属性含像元类型、像元大小（空间分辨率）、位深depth（决定可表示数值范围与精度，不是距离信息也不等于存储容量本身）、空间参考和头文件元数据；矢量与栅格互补：矢量适合离散目标、栅格适合连续场；栅格可通过像元集合表达点线面（单像元点、连续像元线、连通区域面）；常见栅格编码结构有Cell-by-cell逐像元、Run-length游程编码、Quadtree四叉树分解；栅格压缩分无损（可重建，分析优先）与有损（高压缩比，常用于背景显示）；数据转换与集成包括Rasterization（建网格、烧录边界、填充面）和Vectorization（细化、线提取、拓扑重建、平滑去锯齿）。

5. gis data acquisition：地图数据获取主渠道包括GNSS/北斗定位、LiDAR点云、遥感影像、测量与已有图件数字化，不包括气压计作为标准GIS制图主采集手段；遥感按机理分被动遥感（光学、近红外、短波红外、热红外）与主动遥感（SAR），SAR为主动式而非被动式；山火监测常优先红外波段；DEM是规则格网高程数据，属于栅格数据模型（非矢量），常来源于摄影测量立体像对、InSAR、LiDAR；典型数据形态含DOQ正射影像、DRG数字光栅地图、土地覆被栅格、二值扫描图、TIFF/GIF/JPEG及GIS专有格网格式；Terra ASTER、SPOT5等可用于立体测图；卫星差分定位可削弱钟差和部分大气误差，但多路径误差难以通过常规差分消除；导航高程一般先得椭球高，转正常高需大地水准面模型，普通接收机“精确测海拔”表述不严谨；我国北斗在国家应用通常对接CGCS2000坐标框架，不能直接等同“北斗使用WGS84”。

6. geometric transformation：几何变换用于统一多源数据的几何参考，核心流程是控制点匹配->模型求参->误差评估->重采样；常用模型含平移、旋转、缩放、镜像、相似变换、仿射变换（保持直线与平行关系）、射影变换（透视纠正）、多项式/薄板样条（处理局部畸变与橡皮片）；精度评估常看残差与RMSE；栅格重采样方法包括最近邻（保分类值）、双线性（平滑连续）、三次卷积（细节较好但计算高）；矢量变换重在坐标严格映射，栅格变换额外涉及像元重建；工程实践中“先统一基准与投影、后叠加分析”，并结合栅格化/矢量化完成跨模型融合。

真题高频判定与结论并集：1）大地基准“以地心为中心的基准面”表述错误；2）本套题语境下“矢量简单要素不包括Polyline”；3）TIN错误说法是“高差大区域用大三角形”；4）拓扑优点不含“数据结构简单”；5）山火监测优选红外；6）ECEF是地心直角坐标系；7）横轴墨卡托错误说法是“每带中央经线比例因子大于1”或“小于1”为题干指定时应按真题版本辨析（标准UTM取小于1）；8）复合要素不包括Polygon；9）差分定位难消除多路径误差；10）地图数据获取方式不包括气压计；11）“北斗使用WGS84”判错；12）“接收机可精确测海拔”判错；13）TIN是矢量模型判对；14）“Shapefile存储拓扑数据”判错（其为非拓扑简单要素）；15）矢量分区中Polygon通常不允许重叠判对；16）“栅格像元数据类型必须是整型/浮点型”均为绝对化错误，实际可为多种数值类型与分类编码；17）depth不定义距离信息也不直接等同容量信息；18）“SAR是被动遥感”判错；19）“DEM是矢量模型”判错；20）椭球到平面不可能全域无变形，“通过选参数实现无变形投影”判错。
